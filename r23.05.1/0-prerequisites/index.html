Overview

The purpose of this guide is to get you set up with the tools and skills you'll need before diving into MESA. If you're already comfortable with the command line, you can probably skip this module.

This module will cover:

  • Suggested Tools
  • Basic Navigation and Usage of a Shell
  • Setting and Using Environment Variables

Conventions in this Module

As you work through this modules (and others), you'll see a few conventions used to help you navigate the content:

monospaced text with a background
commands you should issue to a shell prompt
monospace text with no background
variables or code that come from a file or that you should enter into a file (context should make this clear)
italicized text like this or this
text you should replace with your own values

Sometimes larger chunks of code will be included in the text, like this:

test_file.py (partial)

{% highlight python linenos %} def hello_world(): print("Hello, world!") if __name__ == "__main__": hello_world() {% endhighlight %}

In this example the code came from or belongs in a file with the same name as the one in the label above the code listing. In this case, that'd be test_file.py. The (partial) label indicates that the code listing is only a partial listing of the file. That is, some or much of the code in that file may have been omitted for clarity.

Depending on the scenario, the listing may call your attention to some code with no action needed on your part, or you may need to compare your code to the provided listing and copy if need be. The context should usually make this clear.
